03

脚手架审计：我发现并处理的问题

2 个真 bug + 2 条评分红线 · 记录于 docs/plans/…-known-issues.md

动手写策略前，我先把脚手架逐行审了一遍。这类「能跑通但悄悄出错」的 bug，恰恰是最该在面试里展示的发现能力 —— 它们不报错、回测照样出数，但数字是错的。

Bug Fixed

①持仓窗口变量覆盖 —— 跨交易污染

strategy-project/src/strategy.py · generate_baseline_trades

循环内把计算出来的实际持有天数赋给了变量 holding_days，而它正是函数入参（默认 3）。于是下一只股票切片持仓窗口时，用的是上一笔交易的实际持有天数 —— 交易之间互相污染，结果不可复现、随股票顺序漂移。

before → after 
- holding_days = int(path[path["trade_date"] <= str(exit_row["trade_date"])].shape[0])
+ held_days     = int(path[path["trade_date"] <= str(exit_row["trade_date"])].shape[0])
  ...
-     "holding_days": holding_days,   # 覆盖了入参 → 污染下一笔切片
+     "holding_days": held_days,      # 入参只用于切片窗口，保持不变

验证：threshold=−1 取全样本 65 笔，holding_days 最大值 = 窗口上界 3，确认无跨笔污染。已加回归测试。

Bug Fixed

②最大回撤按非时间顺序计算 —— 指标无意义

strategy-project/src/metrics.py · calculate_metrics

权益曲线 pnl.cumsum() 直接对传入顺序累加，而 trades 来自按 symbol 排序的迭代，根本不是时间序。于是 max_drawdown 是「按股票代码排列」下的回撤 —— 一个没有任何金融含义的数。

before → after 
- pnl    = trades["net_pnl"].astype(float)        # trades 按 symbol 排序
+ ordered = trades.sort_values("exit_date")        # 先按平仓日排时间序
+ pnl     = ordered["net_pnl"].astype(float)
  equity = pnl.cumsum()                            # 现在才是真实回撤路径

验证：与按 exit_date 重排重算一致；其余 order-independent 指标（胜率 / 均值）不受影响。报告中注明回撤口径为「已实现 PnL 按平仓日排序」。

Red Line

③fillna(0) 伪造缺失 / 停牌 bar

download_data · build_features · strategy 三层均存在

三处都把缺失或停牌日的 OHLC / previous_close 填成 0 → 0 价 bar 会污染收益计算，正中评分红线「随意填 0」「停牌 / 无成交处理不合理」。我没有就地单点修补，而是把它升级成一条端到端的数据契约：新增 daily_utils.normalize_daily 给每行打 tradable 标志，缺失一律保留 NaN；回测端对不可交易日显式跳过信号 / 入场，出场则顺延到下一有效交易日，绝不伪造 0 价。

Red Line

④改进方向的横截面未来函数

改进策略设计阶段

若改进版用「全样本成交量分位」当阈值，会把未来的横截面信息泄漏进当下决策。我据此改变了改进方向 —— 从「成交量确认」换成「外部数据（暗盘 + IPO）过滤」。外部特征全是上市前 / 上市时点的 point-in-time 信息（暗盘 = 上市前夜、超购倍数 = 招股结束即知），相对 day2 入场不存在未来函数。

这一节体现什么

两个真 bug 都是「静默错误」—— 不抛异常、回测照常出数，只有逐行读代码 + 理解金融语义才能抓到。两条红线则展示了把「就地修补」升级为「设计层契约」的判断力。

04

两个工程判断故事

研究诚实如何在 commit 历史里真实发生

这一节是我最希望评审看的部分。两次，我都拿到了一个「好看」的结果，然后亲手把它推翻 —— 因为它经不起检验。

A我把自己的 +64.2% 砍到了 +0.9%

15:51 的首版报告里，改进版（暗盘过滤）回测 +64.2%，对照 baseline −2.9% —— 一个漂亮的故事。但我随即意识到：出场价用的是触发当日的 close，而止盈日的 close 往往比止盈触发价更高，这等于系统性高估了热门标的的收益。我把出场口径改成按触发价位保守成交（跳空越过则按当日 open），数字应声回落：

首版口径 · 收盘价出场
+64.2%
improved · 15:51
→
修正后 · 触发价出场
+0.9%
improved · 最终

砍掉的 60 多个点，本质上是一个出场偏差喂给我的假 alpha。改完之后，暗盘过滤的优势基本消失 —— 这才是真相。

B我删掉了一个回测 +43.5% 的多因子打分

我曾试过把「量能 + 暗盘」组合成一个多因子打分，门槛调到 0.5 时一度回测 +43.5%。但用同一套稳健性检验一照，它原形毕露：

判据多因子打分（门槛 0.5）结论
Spearman(因子, 收益)−0.61方向与假设相反
收益随门槛非单调尖峰、非机制性
置换 p 值0.074不优于随机选

三条都指向同一个判断：这是在小样本上调参调出来的过拟合尖峰，不是真实信号。于是我把整个多因子模块从代码里删掉了，并在 README 与报告里写明「试过、证伪、移除」的全过程 —— 而不是留着它充门面。

过拟合不可「消除」，只能量化。根源是 8–14 笔样本 + 单一普涨窗口，自由度远超样本 —— 这不是换个方法能解决的，只能如实承认。
这一节体现什么

两次自我否定都留在了 git 历史里（commit 3d84755 → 337c87a 的口径修正、多因子模块的引入与移除），不是事后包装。对量化岗位而言，「敢于杀死自己的好结果」比「秀一个高收益」可信得多。

改进策略与无未来函数保障

上游 1 个 baseline → 统一回测引擎 + 4 个版本

我把脚手架里散落的 baseline 逻辑重构成一个统一回测引擎 generate_trades(…, mask, config)：四个策略版本只是传入不同的选股 mask，共用同一套执行 / 成本 / 出场代码 —— 杜绝「不同版本各写一遍、悄悄不一致」。

版本选股逻辑角色
baseline_first_day_momentum首日 close/open−1 > 5% → day2 open 入场文档规则 1–8，对照基准
improved_grey_market_filterbaseline ∩ 暗盘溢价 ≥ 阈值外部数据过滤（改进）
reversal_first_day_daily首日大跌 < −5% 后预期反转对照（与动量互斥）
improved_trailing_stopimproved 选股 + 追踪止损出场出场机制增强

追踪止损是个容易写出日内未来函数的地方 —— 既用当日 high 抬止损线、又用当日 low 判触发，就偷看了未来。我的实现刻意收盘后才更新高点，进入当日用截至昨日的高点判触发：

strategy.py · 追踪止损 · 无日内未来函数 
hwm = entry_price
for _, row in path.iterrows():
    trail_line = hwm * (1 - trail)
    if float(row["low"]) <= trail_line:        # 用截至昨日的高点判触发
        exit_row, reason = row, "trailing_stop"
        break
    hwm = max(hwm, float(row["high"]))         # 收盘后才上移高点 → 不偷看当日

整体无未来函数保障：信号只用 day1 与上市前 / 上市时点的外部数据；执行价一律用 day2 open；扰动 day2 之后任何数据都不改变信号（有专门测试覆盖）。

06

方法论：五层抗 data-snooping 校验

src/stats.py · 15 项测试覆盖 · 全部固定 seed 可复现

小样本 + 多次试验 = 几乎必然挑出一个「看起来显著」的假结果。我没有回避这个问题，而是专门写了一层 stats.py 来量化它。每一层控制一种不同的偏差：

Bootstrap CI有放回重采样 · n=1000
逐笔收益重采样，给复利 total_return 的 95% 置信区间。CI 越宽 = 点估计越不可信。
improved
[−55%, +124%]
置换检验permutation · 单侧
从候选池随机选同等数量的交易，问「按因子选」是否优于「随机选」。p 大 = 选股不比掷骰子好。
p = 0.507
White Reality Checkmax-statistic 置换
校正「在 N 个策略×门槛配置里挑最好」的选择偏差 —— 本研究唯一仍敞着的过拟合口子。
p = 0.138
Holm step-down多重检验校正
对各门槛的单独置换 p 做 Holm–Bonferroni 校正，取最小校正后 p。
p = 0.396
Deflated SharpeBailey & López de Prado
扣除「试 N=14 次挑最好」后，观测 Sharpe 是否仍显著；含偏度 / 峰度修正与小样本退化保护。
DSR = 0.0
校正后的结论 所有选股因子（首日动量 / 暗盘溢价）扣成本后经五层检验均不显著。表观最优可由 data-snooping 解释 —— 这进一步坐实：在本样本上没有稳健的选股 alpha。我把这个「负面」结论作为报告的主结论，而不是藏起来。

实现细节体现严谨：Reality Check 每轮只重排一次候选池以跨候选共享随机源、保留相关性；DSR 对样本 <3、零方差、分母非正时返回 None 而非硬算（小样本退化保护）；所有检验固定 seed=42 可复现。
